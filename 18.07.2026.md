## 2. Что сделано

### Восстановление CRM (главная находка сессии)
Модель `Client`/`Payment`/`Touch` была полностью реализована в сторе и типах, но **без единого пикселя
UI** — ни страницы, ни маршрута, хотя история роадмапа утверждала обратное («страница Клиентов
готова»). Собран заново, специально просто (по прямому ТЗ):
- **`src/pages/Clients.tsx`** — список клиентов (эконом.: оплачено/ожидается/просрочено, бейдж
  «Допродажа» для повторных), простой график выручки по месяцам (`AreaChart`), вкладка **Ученики**.
- **`src/components/ClientDetailDialog.tsx`** — инфо, платежи (отметить оплаченным, добавить),
  привязка/отвязка существующих проектов, заметка.
- **`src/components/StudentsSection.tsx`** — помесячная оплата, деление 1–2 платежа/мес, история с
  датами (выбор через `DatePicker`), статус «скоро оплата»/«оплата проведена»/«просрочено».
- **`src/lib/clients.ts`, `src/lib/students.ts`, `src/lib/payments.ts`** — чистые функции агрегации
  денег и статуса платежа (общий `paymentState()` для клиентов и учеников, окно «скоро» = 3 дня).
- Дашборд: деньги+риск возвращены, затем **убраны обратно** по фидбэку («карточки нахуевертил, верни
  как было») — деньги остались только на странице Клиентов, дашборд снова только про задачи.

### Автономные волны `/loop` (14 штук, каждая с `tsc`+`vitest`+`build`)
Перекат незакрытых задач утром (`OverdueRollover` — **добавлен, затем убран по фидбэку** «зачем ты это
сделал»); вложенные действия в ⌘K на задаче (приоритет/срок, не выходя из палитры); строка
«Создать задачу: …» прямо в ⌘K по набранному тексту; курсор J/K по списку задач (`x` — отметить, `e` —
открыть); расширение NLP (время `в 15:00`→напоминание, `~30м`→оценка, `каждый день`→повтор — подключено
во все 3 места ввода: Dashboard/ProjectDetail/QuickAddDialog); снуз-пресет «на неделю»; починка
устаревшей подсказки «1–8» в диалоге шорткатов (списков 6); чистка отставшего контекст-меню (не хватало
«через неделю»/«убрать срок»).

### AI: из советника в исполнителя
- **`src/lib/aiTools.ts`** (новый) — `create_task`/`complete_task`/`reschedule_task`/`create_project`/
  `create_note` с fuzzy-поиском задачи по подстроке названия (при неоднозначности — сообщает, не гадает),
  проверкой блокировок (`blockedBy`) перед завершением. Каждое действие — через `pushUndo` (тот же
  стек, что Ctrl+Z везде в приложении).
- **`src/lib/ai.ts`** — новый `requestCompletion()` (нестриминговый decide-запрос с `tools`), плюс
  **фикс бага** (см. §3, п.2).
- **`src/components/AiAssistant.tsx`** — decide→tool-exec→wrap-up поток: сначала запрос с `tools`,
  если модель решила действовать — выполняем локально + тост с «Вернуть», затем стримим
  финальный ответ с учётом результата.
- **«Спланируй мой день»** — `aiContext.ts` дополнен оценками времени/приоритетом по задачам,
  дневным лимитом (Иви Ли), параметрами помодоро; новая кнопка-подсказка просит конкретный
  план по времени, а не общие советы.
- **Тесты:** `src/lib/__tests__/aiTools.test.ts` — 15 тестов (создание/поиск/неоднозначность/
  блокировки/undo), без реального API — на фейковом сторе.

## 3. Найденные и исправленные баги (оба — системный дебаг, не догадки)

1. **Краш на каждой загрузке приложения** (`Element type is invalid… Check the render method of
   MobileNav`). Причина: `Sidebar.tsx` и `MobileNav.tsx` держат каждый свою отдельную карту
   `ICONS` по id пункта меню; при добавлении «Клиенты» в `NAV_ITEMS` забыл иконку в обеих картах →
   `<Icon />` с `Icon=undefined`. Починил (иконка `Users` в обеих картах) + написал регресс-тест
   (`src/components/layout/__tests__/nav.test.tsx`), **проверил что тест реально ловит баг** — временно
   откатил фикс, тест упал с той же ошибкой, вернул фикс, тест позеленел.
2. **AI-ключ «слетал»**: сохранение срабатывало только по `onBlur`, что ненадёжно по таймингу.
   Теперь сохраняется на каждое нажатие клавиши (`persistAiIfValid` вызывается из `onChange`, не
   только `onBlur`).
3. **Gemini API 400 `missing thought_signature`**: при повторном запросе (после вызова функции)
   код вручную пересобирал объект tool_call и терял служебные поля, которые Gemini требует вернуть
   дословно. Исправлено — `CompletionResult.rawToolCalls` теперь хранит **сырой** ответ API,
   передаётся дальше без пересборки. `ChatMessage.tool_calls` намеренно типизирован как `unknown[]`
   с комментарием, почему нельзя пересобирать.

## 4. Секреты — как обработаны

Пользователь вставил в чат реальный Gemini API-ключ (`AIzaSy...`). **Ключ не был записан ни в один
файл репозитория** — он живёт только в `localStorage` браузера пользователя (архитектурное решение
проекта: ключ ассистента user-entered, никогда не в сборке/экспорте). В ответе пользователю явно
подтверждено, что ключ не сохранён в код.

## 5. Ограничения безопасности (сохранить дословно, не менялись)

- **Git в этом проекте не трогать** — корневой репозиторий = весь профиль пользователя.
- **Playwright не использовать** — проверка только через `tsc`/`vitest`/`build` + чтение кода;
  находки, требующие визуальной проверки (дебаунс сохранения, размер `IconAction` по всему
  приложению, JSON-экспорт вложений), сознательно отложены, а не сделаны вслепую.
- Все цвета/отступы — через токены ДС, без хардкод-хексов/произвольных `text-[]`/`rounded-[]` и т.д.
- Фильтры — только инлайн, без сворачиваемых панелей.
- Встречи — одна запись, никогда не дублируются как задача.
