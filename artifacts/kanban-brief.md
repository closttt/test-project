# Brief — Kanban board UX deep-dive (for our task manager)

## Продукт
Local-first персональный таск-менеджер + лайт-CRM для одного владельца (фрилансер + репетитор). React + TypeScript + Vite + Tailwind, всё в localStorage, без бэкенда для ядра. Тёмная чистая плоская тема (нейтрально-чёрная, тонкие границы, минимализм). Часто используется **с телефона**.

## Что уже есть в нашем канбане (страница «Задачи», вид «Канбан»)
- 4 режима колонок: **Приоритет** (Высокий/Средний/Низкий/Бэклог), **Проект** (колонка на проект + «Без проекта»), **Статус** (Без срока/Сегодня/Скоро/Готово), **Доска** (кастомные колонки: добавить/переименовать/переставить/удалить, цвет из токенов, лимит WIP с красным счётчиком).
- Карточка: чекбокс, заголовок, приоритет, срок (выпадашка+календарь), потрачено времени, теги, бейдж проекта, замок при блокировке зависимостями.
- DnD: перетаскивание карточек между колонками (HTML5 drag, `dataTransfer`). Клавиатура: стрелки ←/→ переносят карточку в соседнюю колонку, Enter открывает.
- Колонки: горизонтальный скролл, ширина 288px (`w-72`), `overflow-x-auto`.

## Проблема (со слов владельца)
Канбан «сыроват»: мало **мобильности** и **функциональности**. Надо углубить.

## Что исследовать
1. **Мобильный UX канбана**: тач-перетаскивание (HTML5 drag на тач ненадёжен — какие паттерны используют? long-press? кнопки-стрелки? свайп?), горизонтальный свайп между колонками, адаптив ширины колонок на телефоне, «одна колонка на экран» vs скролл, sticky-заголовки, свернуть колонку.
2. **Функциональность доски**: быстрое добавление карточки прямо в колонку («+ карточка»), сворачивание/сворачивание колонок, счётчики/суммы по колонке, свимлейны, фильтры/поиск на доске, **ручной порядок карточек внутри колонки** (не только между), множественный выбор.
3. **Плотность карточки**: компактный режим, что показывать/скрывать, прогресс подзадач, цветовые метки.
4. **Клавиатура и доступность**.

## Конкуренты (брать этих; помечать предположения)
Trello, Linear, TickTick, Notion (board), Height, Jira, Asana, ClickUp, Todoist (board), Microsoft Planner/To Do.

## Целевая аудитория
Один владелец-фрилансер/репетитор, часто с телефона. Ценит скорость, минимум кликов, чистый минимализм. Не нужен энтерпрайз — нужен быстрый, мобильный, приятный канбан.

## Что важно на выходе
Приоритизированная таблица улучшений (must / should / nice) с пометкой у кого из конкурентов есть, + конкретные **мобильные** и **UX** паттерны доски под наш стек (React+Tailwind, без тяжёлых DnD-библиотек по возможности) и нашу плоскую тёмную DS.
