# Бриф: личный таск-менеджер / мини-CRM

## Контекст и продукт

Персональный инструмент для соло-фрилансера/маркетолога: объединяет лёгкую CRM (клиенты + деньги),
управление проектами и задачами, свободные заметки и календарь встреч — в одном простом приложении.

Явно **не** Notion (без универсальных блоков/баз), **не** тяжёлый PM-инструмент с аналитикой и ролями.
Два визуальных ориентира, заданных на старте:
- **Скрин 1** — CRM-дашборд («CRM by Maslov»-подобный шаблон): выручка, календарь, задачи, «зона риска»
  по клиентам, график динамики. Источник вдохновения для дашборда и денежной части клиентов.
- **Скрин 2** — предельно простой Things/GTD-подобный таск-менеджер (Inbox/Важные/Сегодня/Проекты),
  карточки задач с чек-листом подзадач и тегами. Потолок сложности — MVP не должен быть сложнее этого
  по ощущению, даже если по данным богаче.

## Целевая аудитория

Один пользователь — сам владелец продукта. Соло-режим осознанно, без ролей и расшаривания на MVP.

## Ключевые задачи пользователя (JTBD)

1. Открыв приложение, за секунды увидеть, что горит сегодня (задачи + ближайшие встречи).
2. Хранить по каждому клиенту контакты, деньги (выручка / ожидаемые оплаты) и статус — не уходя в Excel.
3. Видеть клиентов «в зоне риска» (просрочка / нет движения) без ручного подсчёта.
4. Группировать задачи по проектам, с чек-листом подзадач внутри задачи (2 уровня вложенности).
5. Держать свободные заметки отдельно от задач и клиентов.

## Цели и критерий «продукт работает»

Качественно: пользователь переходит на этот инструмент вместо текущего разрозненного способа
(заметки/таблицы) и открывает его каждый день как единственную точку входа в работу.
Количественный критерий success не зафиксирован — открытый пункт, вернёмся при роадмапе.

## Ограничения

- **Стек:** React + Vite.
- **UI:** shadcn/ui, без кастомных компонентов и без новых UI-библиотек поверх.
- **Данные:** MVP — локально (localStorage/IndexedDB), без бэкенда и без логина.
  Следующая волна — Supabase (не в MVP).
- **Платформа:** веб, responsive (детали брейкпоинтов — не обсуждались, уточнить при IA/дизайне).
- **Навигация верхнего уровня:** Дашборд (старт), Клиенты, Проекты, Задачи, Заметки, Календарь/встречи.
- **Стартовый экран:** Дашборд-сводка (не список задач).
- **Состав MVP-дашборда:** задачи на сегодня, календарь/ближайшие встречи, клиенты в зоне риска.
  Без графиков и BI-визуализаций.
- **Вложенность задач:** 2 уровня — задача + чек-лист подзадач. Без произвольной глубины.
- **Клиент как сущность:** контакты + деньги (выручка, ожидаемые оплаты) + статус; отдельный раздел
  в навигации (не просто поле у проекта).

## Anti-scope / fail criteria (точно НЕ в MVP)

- Графики / аналитика динамики (finance charts, как «Динамика выручки» на скрине 1).
- Канбан / доска сделок (pipeline-вид).
- Тайм-трекинг и почасовые ставки.
- Уведомления, интеграции (email/calendar sync), вложения файлов.
- Мультипользовательский режим и роли.

## Риски / открытые вопросы

- Язык интерфейса (RU/EN) не подтверждён.
- Связь Проект ↔ Клиент: обязательна или проект может быть без клиента (внутренние задачи)?
- Связь Задача ↔ Проект: обязательна, или задачи могут существовать вне проекта (свободный Inbox,
  как в скрине 2)?
- Правило «клиент в зоне риска» (порог дней без движения / просрочки) не определено численно.
- Количественный критерий «MVP готов и я им реально пользуюсь» не зафиксирован.
