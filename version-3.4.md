# CRM — version 3.4 (компактный хендофф)

Личный таск-менеджер. React + Vite + TS + Tailwind + shadcn/ui. Локал-фёрст (`localStorage`), только десктоп.

## Что нового в 3.4

Проход по фидбэку (Agentation) на 3.3 + переосмысление карточки задачи.

- **Карточка задачи — полностью переосмыслена** (TickTick-стиль, `TaskEditDialog.tsx`): компактная строка
  сверху (срок+напоминание через один дропдаун с календарём, важное, приоритет — `PriorityPicker`),
  крупное название, новое поле `Task.description`, теги (с крестиком прямо на чипе) и проект. Все эти поля
  теперь **мгновенно сохраняются** (как в строке на `/tasks`) — кнопки «Сохранить» больше нет. Подзадачи —
  как было. Ссылки/вложения/комментарии/зависимости/повтор/оценка времени — ушли в компактные значки-
  счётчики в подвале карточки (`FooterIcon`, каждый — свой dropdown).
- **`PriorityPicker`** (новый компонент, `components/PriorityPicker.tsx`) — дропдаун с 4 вариантами вместо
  слепого цикла по клику. Заменил `PriorityFlag`+`onCycle` везде, где приоритет редактируется: строки
  задач (список/канбан), задачи внутри проекта, дашборд («Задачи на сегодня» + быстрое добавление),
  карточки и детальная страница проекта.
- **Календарь как доп. опция** к быстрым «сегодня/завтра/неделя» в дропдауне даты — на `/tasks` (список +
  канбан) и в задачах внутри проекта.
- **Тег убирается прямо на чипе** — крестик на `TaskTag`, без открытия пикера (`components/TaskTag.tsx`).
- **Дашборд**: быстрое добавление задачи — приоритет и дата сразу (дата по умолчанию сегодня); встречи —
  название стало ссылкой на созвон (`Meeting.url`, новое поле), время крупнее; кольцо цели дня уменьшено.
- **Проекты (`/projects`)** — карточки собраны заново: крупная обложка сверху (`ProjectCover`, была
  иконка 36px), без списка задач на карточке (список остался только на странице проекта — так карточка
  не разрастается на больших проектах), архив — кнопка по наведению на обложку.
- **Отменено из 3.3** (прямой фидбэк, не понравилось): правая панель фильтров — вернули инлайн-бар,
  прижатый вправо (`sm:ml-auto`), без сворачивания; приоритет в Базе знаний — убрали полностью (типы,
  Supabase-клиент, SQL-политика, UI).
- **ProjectDetail**: галерея фото (`components/PhotoGallery.tsx` — drag&drop, вставка по ⌘V, лайтбокс) и
  переименование проекта двойным кликом по названию.

**Стоящий `/loop`**: пользователь явно попросил самостоятельный, повторяющийся UX-аудит — каждый цикл
(сейчас каждые 5 минут, начиналось с 10) пишу себе промпт по улучшению продукта и дорабатываю. Находки за
первые 14 кругов (все — в этой же 3.4, не отдельная версия, changelog не плодит запись на каждый мелкий
цикл):

- Клик по названию открывает `TaskEditDialog` теперь **везде**, где задачи показываются списком:
  Задачи/Канбан, Проект, Дашборд (виджет + попап «Открытые задачи»), Календарь (неделя + день), Аналитика
  (2 попапа), Фокус. Раньше половина этих мест умела только галочку.
- Комментарии (и у задачи, и у проекта) редактируются кликом — были только добавить/удалить.
- Приоритет и дата — сразу на этапе создания в глобальном быстром добавлении (⌘K/N) и при создании проекта
  (раньше — только после, кликом на уже созданной карточке).
- Тег на карточке заметки убирается крестиком (`TaskTag`), заметка кликабельна целиком как карточка
  проекта/знаний (была только маленькая иконка-карандаш на hover).
- ⌘K-поиск проиндексировал `Task.description`; поисковая строка появилась в `/archive`.
- **Найдено при само-аудите (`grep` по всем методам стора на нулевое использование в UI)**: ручной таймер
  задачи (`startTimer`/`stopTimer`) был полностью реализован в `DataProvider`, но нигде не вызывался —
  подключил (кнопка в карточке + пульсирующий значок на строке, пока идёт). Шаблоны чек-листов можно было
  создать, но не посмотреть/удалить — добавил карточку в Настройках рядом с шаблонами проектов.
- Пресеты Pomodoro (25/5, 50/10) в `PomodoroSettingsMenu`.
- Самопроверка собственного кода: `<input type="file">` для вложений задачи жил внутри выпадающего меню —
  вынес наружу диалога, чтобы системный файловый диалог не мог закрыть меню раньше onChange.
- ⚠️ Диск C: на машине разработчика в моменте падал до нуля свободного места во время цикла — один `Edit`
  не применился с первого раза (файлы не пострадали). Цикл был приостановлен (`CronDelete`), место
  подтвердили, цикл возобновлён с интервалом 5 минут вместо 10.

Отражено в `src/lib/changelog.ts` (запись "3.4"; запись "3.3" подчищена от двух пунктов, которые сам же
отменил в этом проходе — история не должна врать).
