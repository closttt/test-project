# v2.0 «Глубина» — большой бандл (дизайн-спека)

**Дата:** 2026-07-13
**База:** MVP 1.8 (React 18 + Vite 5 + TS + Tailwind 3 + shadcn/ui, локал-фёрст, localStorage `crm-taskmanager-data-v1`).
**Характер релиза:** 100% локально + **Claude AI** (ключ пользователя, вызовы прямо из браузера). Zoom/Gmail — перенесены на 2.1.

## Цель

Довести личный таск-менеджер до «почти полной рабочей версии»: глубокий тайм-трекинг (помодоро),
отдельная аналитика, усиленный опыт/геймификация, AI-ассистент для отчётов и советов по своим данным,
архив, удобный ресайз дашборда и сквозная клавиатура.

Принцип неизменен: **растём простотой и скоростью, без новых внешних зависимостей.** AI — через
прямой `fetch` к Anthropic API (без SDK-пакета). Графики — существующий кастомный SVG.

## Подсистемы

### 1. Помодоро + тайм-трекинг
- **Модель:** `AppData.pomodoroSessions: PomodoroSession[]` = `{ id, taskId?, kind: "work"|"break", minutes, startedAt(ISO), date(YYYY-MM-DD) }`.
- **Настройки:** `Settings.pomodoro = { workMin: 25, shortBreakMin: 5, longBreakMin: 15, roundsBeforeLong: 4, autostart: false }`.
- **Движок:** глобальный `PomodoroProvider` (context) — держит фазу/остаток/раунд, тикает 1с, живёт над роутером (в `main.tsx`), поэтому таймер не сбрасывается при навигации. По завершении рабочей фазы: `addPomodoroSession(...)`, начисляет минуты в `task.spentMin`, даёт focus-XP, показывает тост, опционально браузерное уведомление.
- **UI:** плавающая панель-таймер снизу справа (`PomodoroBar`): кольцо прогресса, старт/пауза/пропустить/сброс, текущая задача, счётчик раундов. Кнопка «Помодоро» на задаче/в фокус-режиме выбирает активную задачу. Скрыта, если сессия не запущена (только маленький FAB) — не мешает.
- **Интеграция:** старый ручной таймер (`startTimer/stopTimer`) остаётся; помодоро — структурированный режим поверх того же `spentMin`.

### 2. Аналитика (`/analytics`)
- Новая страница + пункт навигации (Sidebar, mobile-nav опц., ⌘K). Только существующие чарты: `AreaChart`, `BarChart`, `DonutGauge`, `HBarList`, `ContributionGrid`.
- Блоки (всё из локальных данных): KPI-строка (закрыто за 30д, часы фокуса за 30д, средний фокус/день, текущий уровень); тренд закрытий (14/30д); часы фокуса по дням (bar); распределение по приоритетам (donut); фокус по дню недели (bar); точность оценок (estimate vs spent, hbar по задачам); топ-проекты по времени (hbar); heatmap фокуса (contribution grid по минутам/день); рост XP.
- Диапазон: переключатель 7/30/90 дней.

### 3. Опыт / геймификация
- `Gamification.bestStreak` реально обновляется (сейчас не пишется). XP за помодоро-сессии.
- Новые достижения: помодоро (первая сессия, 10, 50, 4 подряд без перерыва-пропуска), часы фокуса (10ч всего), аналитик (открыть аналитику), «оценщик» (точность оценки ±20% на 5 задачах).
- Достижения-титулы уровней (напр. 1 «Новичок» … 10 «Мастер потока») — подпись рядом с уровнем.
- Страница «Достижения» дополняется помодоро-статой; level-up-конфетти уже есть — оставляем.

### 4. AI-ассистент (Claude, свой ключ)
- **Ключ:** хранится в **отдельном** ключе localStorage `crm-ai-key` — НЕ в `AppData`, поэтому не попадает в JSON-экспорт. В UI маскируется, есть кнопка «очистить».
- **Настройки:** `Settings.ai = { enabled, model }` (модель — селект; ключ отдельно). Дефолт-модель берём из актуального claude-api (Sonnet как быстрый дефолт, Opus как «умнее»).
- **Клиент `src/lib/ai.ts`:** прямой `fetch` `https://api.anthropic.com/v1/messages`, заголовки `x-api-key`, `anthropic-version`, `anthropic-dangerous-direct-browser-access: true`. Стриминг через разбор SSE (`content_block_delta`/`text_delta`). Без нового npm-пакета.
- **Контекст:** `buildAiContext(data)` — компактный снапшот: сегодня/просрочено/предстоящее, открытые по приоритетам, закрыто за 7/30д, часы фокуса, стрик/уровень, список проектов, заголовки последних заметок. Кладётся в `system`.
- **UI `AiAssistant`:** правый drawer, чат (история в рамках сессии), поле ввода (Enter — отправить, Shift+Enter — перенос), быстрые шаблоны-чипы: «Что делать сегодня?», «Итоги за месяц», «Разбери просроченные», «Совет по фокусу», «Спланируй день». Без ключа — аккуратный экран «добавьте ключ в Настройках». Открытие: кнопка в шапке (`Sparkles`), ⌘K, хоткей.

### 5. Архив
- Поля `archivedAt?: string` у `Task`, `Project`, `Note`. Архивные скрыты из обычных списков/дашборда/фокуса/аналитики-активного, видны на `/archive`.
- Мутации: `archiveTask/unarchiveTask`, `archiveProject/unarchive`, `archiveNote/unarchive`; безвозвратное удаление — существующие `deleteX`.
- Страница `/archive`: сгруппировано (Задачи/Проекты/Заметки), действия «Вернуть» / «Удалить навсегда», undo-тост. Bulk «архивировать завершённые» в задачах.

### 6. Ресайз дашборда
- Починить видимость/юзабельность ручек в режиме «Настроить»: не клипать угловую ручку `overflow`-ом, увеличить и подсветить handle (кастом CSS), включить `e`/`s`/`se`. Подсказка размера при ресайзе не обязательна.

### 7. Клавиатура и команды
- Во всех диалогах: Esc — закрыть, ⌘/Ctrl+Enter — сохранить, focus-trap (у Radix Dialog есть; проверить QuickAdd/QuickNote/TaskEdit). Enter — сохранить в однострочных.
- ⌘K дополняется действиями: «Помодоро — старт/стоп», «AI-ассистент», «Аналитика», «Архив», «Сменить тему», «Выгрузить JSON».
- Новые хоткеи: `g y` → аналитика, `g r` → архив; открытие AI по клавише. `ShortcutsDialog` обновить.

## Инварианты / QA
- `npx tsc --noEmit` чисто; `npx vite build` проходит. Без новых npm-зависимостей.
- Токены DS (tailwind/`index.css`), rem-сайзинг, тёмная/светлая темы, `prefers-reduced-motion`.
- Обратная совместимость: `migrate()` дозаполняет все новые поля; старые бэкапы грузятся.
- Ключ AI не экспортируется и не логируется.
- Нет горизонтального скролла на 1440/768/375; таймер переживает навигацию.
