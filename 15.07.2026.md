## 4. Ошибки и починки

1. **Главный экран не грузился** (`Failed to fetch dynamically imported module`) — dev-сервер vite висел 6+ часов, пока /loop правил файлы. Убил PID 11344, очистил `node_modules/.vite`, перезапустил.
2. **Hermes-бот не стартовал** — модель сама сломала `adapter.py:8338` (кавычки внутри одинарных кавычек в regex). Починил на `r"<a\s+[^>]*href=[\"']..."`, `hermes gateway restart` → `✓ telegram connected`.
3. **Gemini 404 в CRM** — `gemini-2.0-flash` снят Google → `gemini-3.5-flash` в `ai.ts`.
4. **Task не имеет `clientId`** — tsc поймал; убрал linkedTasks, позже сделал транзитивно через проекты. Мой неточный пункт в roadmap исправлен.
5. **Мобильный скролл на 375px** — контролы `flex` без переноса.
6. **Невидимая кнопка под фокусом (16 мест)** — `opacity-0 group-hover:opacity-100` без `focus-visible:opacity-100`.
7. **bulk-complete мимо геймификации** — шёл через `updateTask({done:true})`, не давал XP/стрик/burst.
8. **Uint8Array vs BufferSource** в `push.ts` → `urlBase64ToBytes(): ArrayBuffer`.

## 5. Решённые проблемы (архитектурные)

**Единый источник правды для встреч.** Отказался делать «задачу-копию» — две записи разъедутся. Встреча остаётся одной записью, Задачи рендерят её как строку → «закрыл везде» = свойство архитектуры, а не синхронизация. Добавлен тест-страж:

```ts
expect(result.current.tasks).toHaveLength(0);
```

**Идемпотентность импорта приглашений.** CRM удаляет строку после импорта → дубликат невозможен.

**Эффекты без связи стора и примитивов.** `<html data-effects>` зеркалит настройку.

**Web Push trade-off.** Наружу уезжает только заголовок + время, только для задач с напоминанием; очередь самоочищается. Явно написано в UI Настроек.

**Иви Ли — не переносить данные тихо.** Только визуальный нудж + кнопка.
