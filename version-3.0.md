# CRM — version 3.0 (компактный хендофф)

Личный таск-менеджер. React + Vite + TS + Tailwind + shadcn/ui. Локал-фёрст (`localStorage`), **только десктоп** (адаптив не делаем). `npx tsc --noEmit` чисто, `npx vite build` проходит.

Запуск: `cd "c:/Users/singa/Desktop/Claude/CRM" && npm install && npm run dev` → http://127.0.0.1:5173/ (в этой сессии уже поднят на :5173 и :5174). Сборка: `npm run build`.

## Разделы
`/` дашборд · `/focus` · `/projects` `/projects/:id` · `/tasks` · `/notes` · `/calendar` · `/analytics` · `/archive` · `/settings` · `/achievements` · `/changelog` (Story).

## Что нового в 3.0

Полная дизайн-спека и обоснование (исследование методологий тайм-менеджмента + UX-аудит) — `docs/superpowers/specs/2026-07-13-crm-v3-roadmap-design.md`.

- **Bottom dock** (`components/BottomDock.tsx`) — поиск/добавить/помодоро тремя иконками внизу по центру; из шапки (`AppShell`) убраны Search/Добавить. Помодоро при старте из дока спрашивает задачу (раньше стартовал «без задачи» всегда).
- **Приоритет — цветной чип** (`components/PriorityFlag.tsx`), общий для дашборда/списка задач/канбана/попапов/карточек проекта. В `TaskEditDialog` — 4 цветных кружка вместо `Select` с текстом.
- **Канбан «Доска»** (`lib/kanban.ts`) — четвёртый режим на `/tasks`: свои колонки (add/rename/delete/reorder), персист в `localStorage` (`crm-kanban-columns-v1`). Задачи получили `Task.kanbanColumnId`.
- **Глобальный undo** (`lib/undoStack.ts`) — Ctrl+Z, стек одноразовых restore-функций; те же функции используются тостами «Вернуть» (кто сработал первым — тот и съел запись). Подключено к: удаление/архивация задачи, bulk-действия, заметки, скрытие виджета дашборда, удаление колонки канбана.
- **Вложения к задаче** (`lib/attachments.ts`) — файлы в IndexedDB (`crm-attachments-v1`), метаданные в `Task.attachments: Attachment[]`. UI — в `TaskEditDialog`.
- **Теги задач** сведены к трём: `клиент/личное/работа` (`lib/tags.ts` → `FIXED_TAGS`). Заметки тегируются по-прежнему свободно.
- **AreaChart переписан** (`components/charts/AreaChart.tsx`) — viewBox матчит реальный размер контейнера через ResizeObserver, растяжение исправлено физически, а не косметически.
- **Стат-попапы**: дашборд («Открыто задач», «Закрыто за неделю») и аналитика («Закрыто за N дн.», «Открытые по приоритету») — клик открывает список/разбивку задач.
- **Достижения**: блок «Рекорды» + 15 новых достижений (30 всего), уровни-титулы расширены до 20, level-up получил лёгкий scale/glow-пульс.
- **Проекты**: карточка показывает превью до 4 открытых задач со сроками (не весь бэклог — проекты растут до 100–200 задач, см. roadmap §4 про их будущий отдельный проход).
- **Заметки**: карточки очищены — заголовок, первая строка текста (plain, не markdown-рендер), теги, проект. Дата и полный текст убраны с карточки.

## База знаний из Telegram — стек выбран и запущен (после 3.0, отдельная сессия)
Пользователь подтвердил: **Hermes Agent (NousResearch) через Hermes Desktop** + **Supabase**. Детали и архитектура — спека §5. В коде уже готово:
- `supabase/knowledge_cards.sql` — схема таблицы + RLS (анон-ключ только на чтение).
- `.env.example` → `.env.local` (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`).
- `src/lib/supabase.ts` — REST-клиент без SDK (тот же принцип, что и AI-дизайн v2.0).
- `/knowledge` (сайдбар + ⌘K) — грид карточек, аккуратный экран «Supabase не настроен», пока нет ключей.

Осталось со стороны пользователя: создать проект Supabase, накатить SQL, вписать ключи, установить Hermes Desktop и настроить его Telegram-гейтвей + скилл записи в Supabase — это уже конфигурация внешних инструментов, не код CRM.

## Осознанно не сделано в 3.0 (см. roadmap §4, §6)
- Глубокая переработка «Проекты как центральная сущность» (обложка-картинка, комментарии/лента на проекте, секции как мини-канбан) — пользователь явно попросил отдельный детальный проход, не мешать текущим правкам. Задел есть: механизм вложений и механизм канбан-колонок уже написаны переиспользуемо.
- Полный «TickTick-паритет» канбана (цвета колонок, WIP-лимиты) и полный «фантазийный» пакет геймификации (квесты, сезонные события) — сверх того, что уже выпущено, в беклоге Story.

## Story-роадмап (важное правило — расширено в 3.0)
Раздел Story содержит редактируемый список **«Планируется»** (`lib/roadmap.ts` + `pages/Changelog.tsx`, хранение `crm-roadmap-v1`). **Правило: закрытый пункт СРАЗУ УДАЛЯЕТСЯ, а не отмечается сделанным** — точный `title` в `SHIPPED_TITLES` → `loadPlanned()` авто-удалит его при загрузке.

**Новое в 3.0:** `loadPlanned()` теперь ещё и **авто-мёржит** новые пункты `DEFAULT_PLANNED` в уже сохранённый роадмап пользователя (трекинг «уже показывали» — `crm-roadmap-seen-v1`), ровно один раз каждый. То есть: каждый следующий проход/спека, добавляющая пункты в `DEFAULT_PLANNED`, сама появится в Story при следующей загрузке страницы — сброс вручную больше не нужен. Если пользователь пункт удалил — он помечен «уже видел» и не возвращается.
