# CRM — сводка сессии (compact) от 14.07.2026

> Автосохранённая выжимка из /compact по проекту `Desktop/Claude/CRM` (React + TypeScript + Vite + Tailwind, local-first таск-менеджер/лайт-CRM).

## 1. Запрос и цели
- Продолжить сборку интерфейса по `version-3.2.md`, запустить локально.
- Большой roadmap на 9 пунктов (Insights, недельный виджет, Pomodoro-настройки, инлайн-правки задач, богатые карточки проектов, перенос фильтров, тултипы аналитики, диагностика Hermes/Базы знаний, широкий UX-ресёрч) — согласован целиком.
- Два раунда Agentation-фидбэка со скриншотами, часть решений отменена/скорректирована:
  - Фильтры — только инлайн, без панели-«прятки».
  - В Базе знаний приоритетов быть не должно.
  - Теги убираются прямо на карточке.
  - Календарь дат — доп. опция рядом с быстрыми пресетами, не замена.
  - Приоритет — явный пикер, не блайнд-цикл, везде одинаково.
  - TaskEditDialog — полный редизайн в духе TickTick, но с своей логикой.
  - Projects — card redesign «совсем другой подход» (рефы поручено искать самому).
  - Дашборд: приоритет+дата в быстром добавлении задачи.
  - Встречи: кликабельная ссылка на созвон + крупнее время.
  - Донат-гейдж — уменьшить.
- Постоянные инструкции: работать автономно без лишних вопросов, **не использовать Playwright** (только tsc/build), мыслить масштабнее — самостоятельный аудит роста, упрощение UX, честно предлагать удаление лишнего.
- Финально: `/loop 5m напиши сам себе промпт по улучшению UX продукта и постепенно дорабатывай его с каждым кругом` — рекуррентный автономный цикл улучшений (был остановлен по просьбе, затем перезапущен на 5 минут).

## 2. Что сделано (ключевое)
- **TaskEditDialog** переписан полностью в стиле TickTick: компактная шапка (срок/напоминание/важное/приоритет), крупный тайтл, описание, теги+проект, подзадачи на виду, второстепенное (ссылки/вложения/комментарии/зависимости/повтор/оценка/таймер) — в значках подвала.
- Новые переиспользуемые компоненты: `PriorityPicker`, `TaskTag` (с удалением), `PhotoGallery`, `PromptDialog`, `ConfirmDialog` — заменили все системные `window.prompt`/`window.confirm` кроме двух намеренно оставленных необратимых действий (удаление карточки знаний, полный сброс данных).
- Приоритет-пикер и календарь дат продублированы на всех поверхностях: Tasks (список+канбан), ProjectDetail, Dashboard, Projects.
- Projects — карточки пересобраны: крупная обложка, без списка задач, архив по наведению.
- Клик по названию задачи открывает карточку — теперь единообразно в Tasks, ProjectDetail, Dashboard (оба виджета), Calendar, Analytics (оба попапа), Focus.
- Ручной таймер задачи (был в коде, но нигде не подключён) — кнопка старт/стоп подключена.
- Задачи внутри проекта реально перетаскиваются между секциями (раньше только через select, несмотря на то что в истории значилось «сделано»).
- База знаний: приоритет добавлен и полностью откатён по фидбэку; Hermes/Telegram root-cause диагностирован и передан пользователю (разрыв на стороне кастомного скилла Hermes Agent, не в CRM).
- Фильтры на Tasks/Projects/Knowledge/Notes — везде инлайн, `sm:ml-auto`, без панели.

## 3. Обнаруженные и исправленные баги/несоответствия
- Правая панель фильтров — построена по ресёрчу, отклонена пользователем, откачена.
- Приоритет в Базе знаний — добавлен по ТЗ, но пользователь явно отменил — откачен полностью (types/supabase/SQL/UI).
- «Секции как мини-канбан» в SHIPPED_TITLES было неточным утверждением — задачи нельзя было перетаскивать между секциями, только через dropdown;드래그-н-дроп реализован по факту.
- Диск C: на инциденте `ENOSPC` — на тот момент почти 0 байт свободно; lup был на время остановлен из осторожности. На момент этой сводки — снова здоров (~61 ГБ свободно из ~476 ГБ).
- В отчёте пользователю ранее была ошибка в единицах (спутал МБ/ГБ свободного места) — исправлено явно в следующем цикле.

## 4. Незавершённое / на контроле
- Полная проверка JSON export/import round-trip для новых полей (`Task.description`, `Meeting.url`, `Project.photos`, `Project.priority`) — `exportData()` в `SettingsPage.tsx` использует whole-object спреды (`data.allTasks`, `data.allProjects`), что даёт основания считать округление полным, но трассировка до источника в `DataProvider.tsx` была проверена в последующем цикле (22) — подтверждено чисто, бага нет.
- `/loop` цикл активен на кадансе 5 минут (cron job), продолжает самостоятельно находить и чинить по одной точке роста за круг; отчёты — короткие, по-русски, с фактом typecheck+build.
- Постоянно проверять свободное место на диске перед тяжёлыми операциями сборки.

## 5. Важные пользовательские правила (зафиксированы в памяти)
- Фильтры — только инлайн, без сворачиваемых панелей.
- Playwright для этого проекта не использовать.
- Работать по-крупному, самостоятельно, без лишних уточняющих вопросов — предлагать роадмап шире и внедрять паттерны сквозно.
- Git в этом проекте не трогать (корневой репозиторий — весь профиль пользователя, коммитить небезопасно).
